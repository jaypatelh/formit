User Testing

Introduction

Motivation: The central purpose of our app is to facilitate the process of creating formations. Thus our user study is going to focus primarily on the base screen (the canvas, list of dancers, list of blocks) and how to make those features most intuitive for the user.

Goals and Critical Tasks:
  • testing adding dancers to the canvas
  • testing intuitiveness of assigning blocks
  • testing how to delete dancers from a formation
Driving Questions:
  • What is the most intuitive way to add a dancer to a formation?
  • Are spots created anonymously and then assigned dancers post-creation?
  • Or is it more natural to create a formation already knowing where each specific dancer is going to be?
  • What is the most convenient way to assign blocking to groups of dancer within a formation?

Methods

Study Design: We describe the purpose of the app but not how to use it. Then we give them instructions and observe how they think the app should be working.

Tasks:
  • Create a v-formation with yourself in the center.
  • Now assign blocking to each side of that v-formation.
  • Say that a dancer can’t make it to the show, take them out of the formation.
  • Create a completely new formation now.
  • Assign blocking however you want.
  • Make a formation for 15 people.
  • Draw out the formations from your most recent set.

Discussion

After the first few user tests, we began to see trends in what they liked, disliked, and wished for. For example, labeling the dots with names and the ability to have multiple formations proved to be extremely necessary. Other less obvious features we learned about were having a grid to allow for precise placement, having a way to record what the different blocks were actually doing, and having an “offstage” area for dancers not currently dancing. The “delete selected” button actually proved to be more problematic than helpful and confused many users since the changes were only reflected on the canvas and not on the list of dancers.
Initially, we allowed users to add dancers by tapping the canvas or adding a name to the list. This ultimately caused more confusion and added complexity to the process, so we now only allow dancers to be added if their name is added to the list of dancers.

Implications

Features we have already implemented as a result of user-testing:

  • Dots labelled with dancer names - now users know exactly who is where at all times
  • Multi-select - makes the process of assigning blocking more efficient
  • Save state and multiple formations - expands on the user need, since a given set may have upwards of 20+ formations
  • Snap-to grid - allows users to spend less time on precise placement
  • Automatic de-select - accommodates the users' expectations that after assigning blocking, the dots would de-select
  • Bolded center lines - since most formations are arranged with respect to the center of the stage, indicating where center is makes setting formations easier

Features added to the pipeline as a result of user-testing:

  • Darker shades to indicate “offstage” area - most users ignored the "delete" feature and preferred to drag dancers off to the side when not dancing. That way, they would be able to easily bring the dancer back on stage and be able to keep track of which side of the stage that person was on.
  • Comments/legend for separate blocks - a common complaint was that there was no way to keep track of what each color necessarily meant within each formation
  • Ability to rotate spots with gesture - a common transition used in formation is having dancers rotate spots in a clockwise or counter-clockwise motion. Having this feature would simplify and speed up the process of creating that effect.
  • Compact/expand formations with zoom-in/zoom-out gesture - depending on the number of people in a formation and the stage space available, choreographers need to compact or expand the spacing of a set formation
  • Ability to draw lines to select a line of dancers within the formation - transitions very often are based on the lines within the formation, whether horizontal, vertical, or diagonal, therefore allowing users to select lines quickly would speed up the process of creating new formations
